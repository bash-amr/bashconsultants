Source: `.github/instructions/content-style.instructions.md`.

**Audience:** Denver-metro SMB owners/operators (5–200 emp), time-poor, jargon-skeptical. Care order: ROI, downtime, compliance, staff impact.

**Voice:** confident not boastful; plain English first; contractions OK, no slang/exclamation marks; active voice; lead with business outcome.

**Banned phrases:** "leverage synergies/unlock value/best-of-breed/world-class," "cutting-edge/next-generation/disruptive/revolutionary," "in today's fast-paced world/digital age/AI era," "solutioning/ideate/circle back/low-hanging fruit," vague "empower," and "robust/seamless/scalable" without a backing number. (NOTE: sibling pages cloud.md etc. still contain "leverage"/"scalable" — pre-existing debt, not yet fixed.)

**Formatting rules (high-violation):**
- ONE H1 per page from `title:`; body starts at H2.
- Heading style = SENTENCE CASE ("Cloud architecture and migration"), NOT title case.
- Every acronym spelled out on FIRST use: "Enterprise Resource Planning (ERP)". Watch OCR, ETL, KPI, ARM, HFM, SSIS.
- ONE CTA per page (usually /contact/).

**SEO limits:** title <=60 chars; description 120–155 chars, one sentence, primary keyword; keywords 5–10; >=1 inbound + 1 outbound link per page; external links only to vendor docs/primary sources/regulators.

**Don't claim:** specific certs (AWS SA, MS MVP), headcount, client logos, revenue, "award-winning/industry-leading/best-in-class."

**Locale:** US English, date format "May 18, 2026," USD.
